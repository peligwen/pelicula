# UI Chrome Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Make the bottom decorative footer bar ~50% taller and add a dark-mode night-sky style to the WATCH button.

**Architecture:** Pure CSS changes in nginx/styles.css and nginx/theme.css. No backend, no JS changes. The footer SVG is inlined as a data URI in theme.css; the WATCH button gets a parallel treatment.

**Tech Stack:** CSS, SVG (inlined as data URIs in CSS)

---

## Notes for the implementor

**SVG encoding approach.** All SVGs in this plan are embedded directly as URL-encoded data URIs inside CSS `url("...")` values, matching the encoding style already used by `nginx/theme.css` for the footer: `#` is encoded as `%23`, but `<`, `>`, `'`, `/`, and spaces are left as-is. This is the minimum-escape form that browsers accept in CSS and it keeps diffs readable. **Copy the `url("data:image/svg+xml,...")` blocks verbatim from this plan** — do not "clean up" whitespace or reformat them, and do not replace `'` with `"` inside the SVG (the outer `url("...")` is double-quoted, so the SVG attribute quotes must be single quotes).

**Footer SVG regeneration.** The existing footer SVG has `viewBox='0 0 1440 36'` and would stretch vertically if the footer height grows from 36px to 54px (1.5x). Because `preserveAspectRatio='none'` is in play, the moon (currently `cy='14' r='6'`) and the fog ellipses would end up visibly smeared into ovals. We therefore ship a regenerated footer SVG with `viewBox='0 0 1440 54'`, moon at `cy='21' r='7'`, and all star/cloud coordinates repositioned proportionally. The result renders as a true 1440x54 scene that only stretches horizontally on wider screens (which is fine — the scene is already designed to tile left-to-right).

**Running the dashboard.** The dashboard is served by the nginx container; the fastest loop is `./pelicula up` from the repo root and then visit `http://localhost:7354`. Hard-reload the page (Cmd+Shift+R) after each CSS change since static assets may be cached. If Docker is not running, you can also serve the `nginx/` directory with any static file server (e.g. `python3 -m http.server 8000 --directory nginx`) and open `http://localhost:8000/` — the dashboard JS will fail to fetch API data, but the chrome (footer + WATCH button) renders fine for visual inspection.

**Dark mode toggle.** There is a moon/sun button in the top right (`#theme-toggle`). Click it to flip between light and dark. The selector pattern pelicula uses is paired: one rule under `@media (prefers-color-scheme: dark) { :root:not([data-theme="light"]) ... }` (OS-level dark, unless user forced light) and one rule under `html[data-theme="dark"] ...` (user explicitly picked dark). Every dark-mode chrome rule in `theme.css` ships as this pair — the WATCH button rule added in this plan follows the same pattern.
