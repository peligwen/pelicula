# Pelicula Roadmap

Pelicula's core phases (A–F) are shipped. This file tracks what's next, what's deferred, and summarises what landed.

---

## Active

Rolling section-review sweeps are the active work mode — each round picks
one cohesive surface, audits for drift / latent defects / hardening gaps,
and ships 3-6 phases as logical commits with tests. The per-round entries
in `## Shipped` capture each pass.

---

## Deferred

- **Invite Apprise notification**: notify admin via Apprise/internal feed when an invite is claimed. Deferred; low priority since the dashboard shows active invites and redemption history.

- **Test-stack durability beyond ~24h**: a `pelicula test --keep` stack left running ~24h wedges with `SQLite Error 14: 'unable to open database file'` from Jellyfin's user store, causing `/api/pelicula/auth/login` to return 503 and blocking every authenticated Playwright spec. Targeted `docker restart pelicula-test-jellyfin-1` doesn't recover (DB returns 404 for the seeded admin after restart — likely a state-rehydration gap). Workaround today is `down -v --remove-orphans` + fresh `bash tests/e2e.sh`. Worth investigating whether the Jellyfin SQLite WAL is being corrupted by container clock drift, or whether a periodic config-seed refresh would keep the admin user pinned. No timeline.

- **Retire/retention/storage pruning**: storage management and dedup reporting. Deferred, no timeline.
- **NFS-backed library (named volumes)**: host `movies/` and `tv/` on a NAS via NFS without a macOS Finder mount. Docker Desktop's Linux VM mounts the export directly through `local` volumes with `driver_opts: type=nfs`, so containers read/write it as normal named volumes — no `/Volumes`, no VirtioFS, no FUSE. Keep `WORK_DIR` (downloads + processing) local because NFS breaks hardlinks and is poorly suited to active torrent I/O; accept that Sonarr/Radarr will fall back to copy-on-import. Shape: new `compose/docker-compose.nfs.yml` + `compose/docker-compose.local-library.yml` override pair; `LIBRARY_NFS` / `NFS_HOST` / `NFS_EXPORT` / `NFS_OPTIONS` in `.env`; ``pelicula up`` picks the right overlay. Full plan: `~/.claude/plans/shiny-floating-cosmos.md`.

### 2026-05-02 — typed-client migration sweep (R18)
- Round 16.5 threaded ctx through ~190 legacy `services.Clients` callsites but left the underlying http.Client pool intact — the legacy `arrDo`/`ArrGet`/`ArrPost`/`ArrPut`/`ArrDelete`/`ArrGetAllQueueRecords`/`QbtGet`/`QbtPost` helpers used a plain *http.Client (10s timeout, UA only) instead of the R9 typed *httpx.Client (selective retry on 408/429/5xx, body-drain, structured error wrapping with secret redaction). R18 retires the legacy helpers entirely; every API call to *arr/qBittorrent now goes through the typed Sonarr/Radarr/Prowlarr/Qbt clients.
- Thread-safe typed-client reload (P1) — the prior code did `c.Sonarr = arrclient.New(...)` under `c.mu.Lock()` while concurrent handlers read `svc.Sonarr.X(...)` with no lock — a data race per the Go memory model. Fixed by making `httpx.Client.APIKey` private behind an internal RWMutex with a thread-safe `SetAPIKey` setter; `arr.Client.SetAPIKey` and `bazarr.Client.SetAPIKey` delegate. `services.New()` now constructs all five typed clients (Sonarr, Radarr, Prowlarr, Bazarr, Qbt) exactly once; `ReloadKeys()` mutates via SetAPIKey instead of replacing pointers. Five accessors (`SonarrClient/RadarrClient/ProwlarrClient/BazarrClient/QbtClient`) return the published-once pointers. autowire's `ArrSvc` gains `SetBazarrAPIKey(string)` (replaces the old `SetBazarrClient(string, *Client)` shape); the dead nil-check on `BazarrClient()` post-construction was also removed. Tests pin pointer-stability across ReloadKeys, accessor==field identity, and race-free reload while requests are in flight (load-bearing under `go test -race`).
- Enriched arr.Client (P2) — added 12 typed methods + 14 tests to cover every URL shape that legacy callsites hit: LookupMovie/LookupMovieByTmdbID/LookupSeries (search → add flows), GetAllQueueRecords (paginates GetQueuePage with pageSize=100, replaces the legacy ArrGetAllQueueRecords contract), UpdateNotification/UpdateDownloadClient (symmetry with existing Add*), ListReleaseProfiles/AddReleaseProfile/UpdateReleaseProfile (autowire), GetHistory/GetMissing (raw []byte to preserve caller-side ad-hoc parsing), ListIndexers (Prowlarr startup probe).
- Library/catalog/search/missingwatcher migration (P3) — ~52 callsites across catalog, library, search, missingwatcher migrated to typed clients. Consumer interfaces refactored: catalog.ArrClient/library.ArrClient/search.ArrClient drop Arr* methods and gain SonarrClient/RadarrClient/ProwlarrClient typed accessors; missingwatcher uses *services.Clients directly. catalog.RunQueuePoller drops the radarrURL/sonarrURL params (typed clients carry their URLs internally) — supervisor call site updated. Tests reworked: stubs that intercepted ArrGet now use httptest.NewServer with arr.New(srv.URL, "key") for true end-to-end ctx propagation; the R16.5 ArrFulfiller_PassesCtx test redesigned to capture r.Context() at the inbound HTTP request rather than at the legacy interface boundary.
- Downloads + sse migration (P4) — ~14 qbt/arr callsites in downloads.go and sse/poller.go migrated. downloads.QbtClient interface renamed to Svc, drops 9 legacy method shapes for Keys+SonarrClient+RadarrClient+QbtClient. removeFromArrQueue and unmonitor helpers take *arr.Client + apiVer instead of (baseURL, apiKey, apiVer); HandleDownloadPause dispatches StartTorrent/StopTorrent directly instead of computing endpoint+form strings. HandleDownloads parses qbt.Torrent/qbt.TransferInfo typed returns. sse.ServiceQuerier swaps QbtGet for QbtClient(); fetchDownloads uses qbt.RawGet to preserve transparent SSE forwarding semantics.
- Backup/hooks/sysinfo/autowire/bootstrap/app migration (P5) — final ~30 callsites across the remaining packages. backup/hooks/sysinfo consumer interfaces drop ArrGet/ArrPost; gain SonarrClient/RadarrClient. autowire.ArrSvc drops ArrGet/ArrPost/ArrPut/ArrDelete and gains SonarrClient/RadarrClient/ProwlarrClient — the autowire helpers (wireDownloadClient, wireRootFolder, wireReleaseProfile, wireImportWebhook, wireProwlarrApp, triggerHealthCheck) take *arr.Client + apiPath instead of (baseURL, apiKey, apiPath). hooks.Handler drops the ArrGet func-value field; gains SonarrClient/RadarrClient *arr.Client fields wired from svc.SonarrClient()/svc.RadarrClient() in bootstrap. bootstrap's catalog cache closures use catalogCacheSvc.RadarrClient().Get / SonarrClient().Get for raw byte semantics. app.go's prowlarr indexer probe uses ProwlarrClient().ListIndexers. arr.Client.Add*/Update* signatures relaxed from `map[string]any` to `any` so autowire can pass typed structs without conversion. The backup TestHandleExport_RequestsCtxCancelled test split into two: TestHandleExport_IncludesRequests (positive integration) and TestRequestStoreAll_CancelledCtx (lower-layer ctx contract) — the typed-client path now correctly fails fast on cancelled ctx instead of silently returning empty arrays.
- Legacy helpers retired (P6) — deleted `arrDo`/`ArrGet`/`ArrPost`/`ArrPut`/`ArrDelete`/`ArrGetAllQueueRecords`/`QbtGet`/`QbtPost` from `services/clients.go` along with the now-unused `redactedURL` helper (it had zero callers; the typed clients have their own redaction in error messages). Kept `c.client *http.Client` (renamed comment, unchanged code) — it's still load-bearing for `CheckHealth`'s seven-backend parallel probes (typed clients would retry 3× on 5xx and exceed the 2s per-probe budget) AND it's exposed via `HTTPClient()` to four production consumers: bootstrap (Jellyfin + Procula NewWithHTTPClient + actions handler), autowire (waitForServices polling), jellyfin/wiring.go, and cmd/pelicula-api/jobs.go (procula jobs proxy). Documented its single load-bearing role in code. Round 16.5's "future round candidate" note is now closed: the legacy services.Clients raw HTTP pool no longer carries any deprecated helper API surface.
- ~270 callsites migrated total across P3-P5; ~30 new ctx-propagation/race tests + 14 typed-method tests added; full middleware test suite (49 packages) green at finish under `go test ./...` and `go test -race ./internal/app/services/...`.
