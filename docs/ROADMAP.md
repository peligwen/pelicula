# Pelicula Roadmap

Pelicula's core phases (A–F) are shipped. This file tracks what's next, what's deferred, and summarises what landed.

---

## Active

_Nothing in active development — v0.1 scope is complete._

## Deferred

- **Invite Apprise notification**: notify admin via Apprise/internal feed when an invite is claimed. Deferred; low priority since the dashboard shows active invites and redemption history.

- **Retire/retention/storage pruning**: storage management and dedup reporting. Deferred, no timeline.
- **NFS-backed library (named volumes)**: host `movies/` and `tv/` on a NAS via NFS without a macOS Finder mount. Docker Desktop's Linux VM mounts the export directly through `local` volumes with `driver_opts: type=nfs`, so containers read/write it as normal named volumes — no `/Volumes`, no VirtioFS, no FUSE. Keep `WORK_DIR` (downloads + processing) local because NFS breaks hardlinks and is poorly suited to active torrent I/O; accept that Sonarr/Radarr will fall back to copy-on-import. Shape: new `compose/docker-compose.nfs.yml` + `compose/docker-compose.local-library.yml` override pair; `LIBRARY_NFS` / `NFS_HOST` / `NFS_EXPORT` / `NFS_OPTIONS` in `.env`; ``pelicula up`` picks the right overlay. Full plan: `~/.claude/plans/shiny-floating-cosmos.md`.

**Peligrosa Robustness & Audit (2026-05-01):** Five-commit pass over `middleware/internal/peligrosa/` (~4,649 LOC across 19 files), the user-interaction safety layer covering auth, sessions, login rate limiter, loopback auto-session, open registration, CSRF origin guards, invites, viewer request queue, and role mapping. Invite rate-limit gaps (`01a86f2`): `HandleInviteCheck` was a method on `*InviteStore` with no rate-limit wired in — the public `GET /api/pelicula/invites/{token}/check` endpoint exposed an unbounded 200/404/410 oracle for token probing. Receiver moved onto `*Deps` (mirroring `HandleInviteRedeem`'s pattern), `p.Auth.isRateLimited` check added, and `recordFailure` now fires on the not-found branch (a probe miss is brute-force evidence) but not on the not-active branch (a revoked or expired token is one valid lookup). In `HandleInviteRedeem`, `recordFailure` was previously only in the catch-all error branch — the canonical brute-force signals (`ErrInviteNotFound`, `ErrInviteNotActive`) had no failure accounting, while infrastructure errors (Jellyfin 5xx) WERE tripping the limiter and could lock out legitimate retrying viewers during an outage. Moved the increment to the not-found / not-active branches and removed it from the catch-all. Three tests added (`TestInviteCheck_RateLimit`, `TestInviteRedeem_NotFound_Counts`, `TestInviteRedeem_InfraError_DoesNotCount`), all using `newTestJellyfinAuth` so the SQLite-backed limiter actually exercises. Approve TOCTOU + double-add (`7b2f255`, `b2c24b0`): `handleRequestApprove` had a check-then-act race between the pending-state gate, the `AddMovie`/`AddSeries` network call, and the unconditional state UPDATE. Two concurrent admin approvals (or a click-spam) both passed the gate, both called the *arr fulfiller (creating duplicate Radarr/Sonarr entries), and both wrote `grabbed`. Two layers of defense: `RequestStore` gained an unexported `approveMu sync.Mutex` acquired at the top of the handler — admin approvals are infrequent so global serialization is fine and far simpler than per-id sharding — and `internal/repo/requests/Store` gained `MarkGrabbedIfPending(ctx, id, arrID, updatedAt) (bool, error)` performing a state-conditional `UPDATE … WHERE id=? AND state='pending'` with `RowsAffected` discrimination as second-line defense. The post-`!ok` path now logs `slog.Warn` for the orphan-*arr-add condition (the network call already succeeded but the row left pending), and a failed response re-fetch after a successful state transition degrades gracefully to a minimal `{id, state, arr_id}` JSON body instead of a confusing 404. `TestApprove_ConcurrentRace` asserts exactly one 200 / one 409 / one fulfiller invocation / one `grabbed` audit event, passing 10/10 under `-race`. Atomicity + log redaction (`d9ff391`): `MarkAvailable` and `HandleRequestDeny` previously wrote a state UPDATE then a separate audit-event INSERT — a crash between left a request whose state changed but with no event, requiring admin reconciliation. New `(*Store).UpdateAndInsertEvent(ctx, req, ev) error` in the repo runs both writes inside a single SQL transaction (BeginTx + deferred Rollback + RowsAffected==0→ErrNotFound + INSERT + Commit). Used at both call sites; `handleRequestApprove` deliberately not folded in because its conditional `MarkGrabbedIfPending` CAS would be silently weakened by an unconditional UPDATE. Three default profiles ship enabled (Compatibility 1080p, Compatibility 720p, Downscale 4K to 1080p) — they remain inert until the user opts into transcoding via `TRANSCODING_ENABLED=true`.

**Phase F — External Notifications (Apprise):** Apprise container (opt-in Docker Compose profile), `direct` mode for single-webhook setups (ntfy, Gotify, any webhook URL), config at `/config/procula/notifications.json`. Discord is not a supported provider.

**Invite Flow (Phase D follow-up):** One-time shareable invite links for admin-free user onboarding. `POST /api/pelicula/invites` generates a 32-byte random base64url token (stored in `/config/pelicula/invites.json`) with configurable TTL (default 7 days) and optional max-uses cap. Admins create links via the dashboard Users section ("Create invite link" button). `/register` (static HTML, no Pelicula session required) renders a username+password form that submits to `POST /api/pelicula/invites/{token}/redeem`, creating the Jellyfin account on success. Expired, exhausted, and revoked tokens return clear error states. Admin can revoke or delete tokens from the dashboard.

**Bazarr — Subtitle Acquisition:** Bazarr container wired into the stack alongside Sonarr/Radarr. Auto-wire creates a language profile from `PELICULA_SUB_LANGS` (set via `the Settings UI` → Subtitles) and connects Bazarr to both *arr apps on startup. nginx proxies `/bazarr`. Procula flags imports with missing subtitles, then actively kicks Bazarr via its API to search immediately (rather than waiting for Bazarr's background polling cycle), waits up to 30 minutes for sidecars to appear, and proceeds to dual-sub generation once they arrive.

**Dual Subtitles:** Procula pipeline stage that generates stacked ASS sidecar files (`Movie.en-es.ass`) alongside source media. Base language (familiar) appears bottom-center in white; secondary (learning) appears top-center in yellow. Source cues are extracted from embedded subtitle streams or `.{lang}.srt` / `.{lang}.ass` sidecars; Argos Translate (offline, not bundled in image) synthesizes missing tracks. Cue alignment is base-anchored (secondary cue midpoint must fall within base cue range). Configurable via env vars (`DUALSUB_ENABLED`, `DUALSUB_PAIRS`, `DUALSUB_TRANSLATOR`) and Procula settings UI. Known limitations: no bitmap (PGS/DVD) sub support; font fallback required for Arial; per-title opt-out not yet implemented.
