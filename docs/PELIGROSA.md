# Peligrosa — User Interaction Safety Layer

Peligrosa is the conceptual layer for every place Pelicula touches input from outside the operator trust boundary: authentication, user management, invite redemption, viewer requests, incoming webhooks, the folder browser escape guard, and CSRF origin checks.

The name is intentional: **peligrosa** (Spanish, *dangerous*). Every piece of code here is a place where the system could be abused.

---

## Threat Model

**Pelicula is LAN-only on port 7354.** The threat model is **"trusted LAN + invitees with their own devices."** Port 7354 is not hardened for public internet exposure and must not be port-forwarded.

**Jellyfin external access:** Jellyfin's built-in HTTPS on port 8920 is the only internet-exposed surface. Pelicula publishes 8920 on the host; the operator handles port-forwarding. Pelicula does not orchestrate certs — Jellyfin handles its own TLS. See [Exposing Jellyfin Externally](#exposing-jellyfin-externally).

**Why auth and CSRF still matter on a trusted LAN:** Invitees browse the web from the same network. A compromised device, or a phishing link opened on a kid's iPad, could attempt a cross-origin request against `http://<nas>:7354/`. Auth sessions, CSRF origin checks, and invite tokens defend against that — not a network intruder, but a confused or tricked client.

**Out of scope:** multi-tenant hardening; host compromise defense; torrent traffic protection beyond the VPN kill-switch.

## Authentication and Sessions

Auth is always on. Credentials are verified against Jellyfin's `/Users/AuthenticateByName` — no passwords stored by Pelicula. Roles in `pelicula.db` (`roles` table); Jellyfin admins automatically get `admin` role.

Sessions are hybrid: in-memory map for fast lookup, DB-backed (`sessions` table) so logins survive a middleware restart. `pelicula_session` HttpOnly cookie, `SameSite=Strict`, 24-hour lifetime. Login rate limiter: 5 failures per IP in 5 minutes → HTTP 429, DB-backed so it persists across restarts. Guards: `Guard` (viewer+), `GuardManager` (manager+), `GuardAdmin` (admin), wired per-route in `main.go`.

**Loopback auto-session** (`loopback.go`): requests from the host machine get a transient admin session without a cookie; three gates all must pass: (1) `RemoteAddr` within `172.16.0.0/12` (Docker bridge), (2) loopback `X-Real-IP` (nginx rewrites this from `$remote_addr` unconditionally — LAN clients cannot spoof it), (3) loopback `Host`. LAN clients must authenticate normally. Override the upstream CIDR with `PELICULA_UPSTREAM_CIDR` in `.env` if needed.

## Users, Invites, and Requests

**Open registration** (optional): `PELICULA_OPEN_REGISTRATION=true` in `.env` enables `/register` without an invite token. Rate-limited; LAN-only via `requireLocalOriginStrict`.

**Invites**: 32-byte crypto/rand token, base64url-encoded, stored in `pelicula.db`. States: active / revoked / expired / exhausted. Redemption creates a Jellyfin user at `viewer` role. Public endpoint gated on valid token; admin endpoints are admin-only.

**Request queue**: viewers submit media requests from dashboard search; admins approve/deny with Radarr/Sonarr quality profiles. Trust split is structural in the route table: `GET/POST /api/pelicula/requests` requires viewer+; `POST/DELETE /api/pelicula/requests/{id}` requires admin.

## CSRF Origin Guard

`isLocalOrigin(origin)` returns true for RFC1918/localhost Origins, parsing as a URL to prevent substring bypasses. Two wrappers wired per-route:
- **`requireLocalOriginStrict`** — rejects state-mutating requests (POST/PUT/PATCH/DELETE) with empty or non-local Origin. Used for admin endpoints.
- **`requireLocalOriginSoft`** — allows empty Origin (API/curl) but rejects non-empty non-local Origins. Used for endpoints with legitimate programmatic callers.

## Webhook Secret and Folder Browser

**`WEBHOOK_SECRET`** (generated by setup wizard): `wireImportWebhook()` injects it as `X-Webhook-Secret` into Radarr/Sonarr webhook configs. `HandleImportHook` verifies with `crypto/subtle.ConstantTimeCompare`. Import webhook endpoint additionally restricted to Docker-internal networks in `nginx.conf`. Incoming webhook payloads are path-allowlisted against known media roots before forwarding to Procula.

**Folder browser** (`library/handleBrowse`): resolves symlinks via `filepath.EvalSymlinks`, then re-checks against allowlist roots. Callers cannot escape configured browse roots through symlinks.

## Exposing Jellyfin Externally

Pelicula does not manage remote access or TLS. To expose Jellyfin:

1. **Enable HTTPS in Jellyfin admin** — Networking → enable HTTPS on port 8920. Jellyfin auto-generates a self-signed cert; you can supply your own.
2. **Port-forward 8920** on your router or DSM firewall to the NAS.
3. Clients connect to `https://<external-ip>:8920/`. Jellyfin and TV apps accept self-signed certs; browsers will warn unless you use a CA-signed cert.

DSM's built-in reverse proxy (Application Portal → Reverse Proxy) can terminate TLS in front of Jellyfin with a Let's Encrypt cert if preferred.

**Invitee dashboard access:** For invitees who want to reach the Pelicula dashboard without exposing port 7354, [Tailscale](https://tailscale.com/) is the simplest path — install it on the NAS and on invitees' devices. They'll reach the LAN dashboard normally. Not a Pelicula feature; an operator choice.

## Known Limitations

- WireGuard private key and API keys in `.env` are plaintext on the host. `pelicula up` sets `chmod 600`; anyone with host access can read them.
- Auth rate limiter is per-IP, DB-backed, persists across restarts. Protects against online brute force only.
- Invite tokens require a DB lookup for validity. 256-bit entropy makes enumeration infeasible.
- `WEBHOOK_SECRET` is optional for backward compat. Fresh installs get a random secret from setup.
- CSRF guards are per-route wrappers — state-mutating admin ops require both a valid admin session and a local Origin.
