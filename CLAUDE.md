## About This Repository

**Pelicula** is a clone-and-run media stack for macOS and Synology NAS. The `./pelicula` bash CLI handles setup, lifecycle, and health checks for a Docker Compose stack of 8 services behind an nginx reverse proxy on port **7354** (PELI on a phone keypad).

## Key Files

- `pelicula` — single-file bash CLI, all subcommands and platform detection
- `docker-compose.yml` — parameterized with `${CONFIG_DIR}` and `${MEDIA_DIR}` env vars
- `docker-compose.override.yml` — generated on Synology only (adds TUN device mapping), gitignored
- `middleware/` — Go backend (auto-wiring, unified search, download dashboard, auth)
- `nginx/nginx.conf` — reverse proxy config, path-based routing to all services
- `nginx/index.html` — dashboard with telemetry, search, downloads, service status
- `.env` — generated by `./pelicula setup`, gitignored
- `synology-media-stack.md` — original detailed reference documentation

## CLI Commands

```
./pelicula setup       # interactive .env generation + TLS cert + directory creation
./pelicula up          # seed configs, start stack, wait for VPN, print URLs
./pelicula down|status|logs [svc]|update|check-vpn
```

## Architecture

Eight Docker containers via Docker Compose:

```
nginx (:7354) ─── /                → dashboard (static HTML)
               ── /api/pelicula/   → pelicula-api (Go middleware)
               ── /api/vpn/        → gluetun control API
               ── /sonarr/         → Sonarr
               ── /radarr/         → Radarr
               ── /prowlarr/       → Prowlarr
               ── /qbt/            → qBittorrent (via gluetun network)
               ── /jellyfin/       → Jellyfin (NOT behind VPN)
```

**Go middleware (`pelicula-api`)** runs on port 8181 internally, proxied at `/api/pelicula/`. On startup it:
1. Reads API keys from mounted *arr config.xml files
2. Waits for all services to be healthy
3. Auto-wires: adds qBittorrent as download client in Sonarr/Radarr, sets root folders, connects Prowlarr to both
4. Serves unified search (parallel Sonarr + Radarr lookup, interleaved results), download status from qBittorrent API, and optional session auth

**Config seeding (`cmd_up` in pelicula script):** On first run, seeds config files before containers start:
- *arr apps: UrlBase + auth disabled for local addresses
- qBittorrent: subnet auth bypass for Docker network (172.16.0.0/12)
- Jellyfin: BaseUrl

**Critical networking detail:** qBittorrent runs on the `gluetun` Docker network (`network_mode: "service:gluetun"`). nginx and the *arr apps reach it at `gluetun:8080`. The Docker subnet auth bypass means no password is needed for internal service-to-service communication.

## Platform Detection

The `detect_platform()` function checks for `/proc/syno_platform` or `/volume1` to determine Synology vs Mac. This affects:
- Whether `sudo` is prepended to docker commands
- Default paths (`./config` + `~/media` on Mac vs `/volume1/...` on Synology)
- Whether `docker-compose.override.yml` is generated (TUN device mapping)

## Key Constraints

- **Gluetun** is pinned to `v3.41.0` — the `latest` tag tracks an unstable dev branch
- **ProtonVPN requires a paid plan** (Plus or higher) — free tier lacks P2P and port forwarding
- **Do NOT enable "Moderate NAT"** when generating the Wireguard key — incompatible with port forwarding
- LinuxServer.io images are Alpine-based — healthchecks use `wget` (not `curl`)
- All volume paths in docker-compose.yml are env vars — never hardcode paths
- **Self-signed HTTPS breaks Chrome** — Chrome blocks JS on self-signed cert pages. Default to HTTP for LAN.
- The middleware has no external Go dependencies (stdlib only) — `go.mod` has no requires
